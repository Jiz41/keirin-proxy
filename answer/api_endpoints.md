## `index.js` APIエンドポイント一覧

`index.js`には、以下の4つのAPIエンドポイントが定義されています。

### 1. `GET /`

*   **目的:** サーバーのヘルスチェック（死活監視）用エンドポイント。
*   **クエリパラメータ:** なし。
*   **正常時のレスポンス:**
    *   ステータスコード: `200 OK`
    *   ボディ: `ok`という文字列。
*   **説明:** サーバーが正常に起動していることを確認するために使用されます。

### 2. `GET /kaisai`

*   **目的:** 指定された日付の競輪開催情報を取得します。
*   **クエリパラメータ:**
    *   `date` (必須): 開催日を`YYYYMMDD`形式の文字列で指定します (例: `20240522`)。
*   **正常時のレスポンス:**
    *   ステータスコード: `200 OK`
    *   ボディ: `kaisai.js`の`getKaisai()`関数から返される、開催情報を含むJSONオブジェクト。
*   **異常時のレスポンス:**
    *   `date`パラメータが指定されていない場合、ステータスコード`400 Bad Request`とエラーメッセージを返します。
    *   処理中にエラーが発生した場合、ステータスコード`500 Internal Server Error`とエラーメッセージを返します。

### 3. `GET /race`

*   **目的:** 指定されたレースIDのレース詳細情報（オッズなど）を取得します。
*   **クエリパラメータ:**
    *   `raceId` (必須): レースを一意に識別するID。
*   **正常時のレスポンス:**
    *   ステータスコード: `200 OK`
    *   ボディ: `scraper.js`の`scrapeRace()`関数（現在は`getOdds`が実質的な処理を担当）から返される、レースデータを含むJSONオブジェクト。
*   **異常時のレスポンス:**
    *   `raceId`パラメータが指定されていない場合、ステータスコード`400 Bad Request`とエラーメッセージを返します。
    *   処理中にエラーが発生した場合、ステータスコード`500 Internal Server Error`とエラーメッセージを返します。

### 4. `GET /debug`

*   **目的:** 特定のURL(`https://keirin.kdreams.jp/komatsushima/racedetail/7320260310020001/`)をスクレイピングし、中間的な解析結果をJSON形式で返すデバッグ用のエンドポイント。
*   **クエリパラメータ:** なし。
*   **説明:** このエンドポイントは、本番運用を目的としたものではなく、開発中のスクレイピングロジックの検証やデバッグのために使用されるものと推測されます。
